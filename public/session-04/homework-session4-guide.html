Step 1 — Setup

Get Your API Key & Connect It

Your agent needs an API key to talk to Google's Gemini AI. Think of it as a password that lets your code use the AI service. It's free — takes about 2 minutes.

🔑 Part A — Create your key

  1. Open a new browser tab and go to: aistudio.google.com/apikey
  2. Sign in with the same Google account you used for Colab.
  3. Click the blue “Create API Key” button.
  4. If asked to choose a project, select any one or “Create project.”
  5. A long string of letters and numbers appears — this is your key. Click the copy icon next to it.

🔒 Part B — Add the key to Colab

  1. Go back to your Colab notebook tab.
  2. In the left sidebar, click the key icon (the “Secrets” panel).
  3. Click “Add new secret.”
  4. In the Name field, type exactly: GOOGLE_API_KEY — all caps, underscores, no spaces.
  5. In the Value field, paste your API key (Ctrl+V or Cmd+V).
  6. Toggle “Notebook access” to ON (it turns blue/green).

✅ Part C — Test the connection

  1. Scroll to the top of your notebook. Find the first gray box with code (under Part 1 — Setup).
  2. Run the cell: click the play button (▸) on its left, or press Shift+Enter.
  3. Run the next cells in Part 1 one by one. You should see: Gemini is ready!
🎉 You're all set if

You see a success message with no red errors. The AI is connected and waiting for instructions.

🚨 If you see an error
  • API key not found → Secret name must be exactly GOOGLE_API_KEY with “Notebook access” ON.
  • Invalid API key → You copied only part of the key. Go back to AI Studio, copy the full key.
  • Permission denied → Make sure AI Studio and Colab use the same Google account.

🧪 Question 1: Tool descriptions matter

Go back to your tool definition. Change the description to something vague: "A tool that does stuff". Run one of your test queries again. What happens? Change the description back. In your text cell, explain what happened and why description quality matters.

💡 This is a real experiment

When you change the description to something vague, the agent will likely either (a) not call your tool at all, or (b) call the wrong tool. That's the point — the description is literally how the AI decides which tool to use. Like a job posting: if it says "do stuff," nobody knows whether to apply.

Troubleshooting

FAQ & Common Issues

Click any question to expand the answer.

🚨 I get NameError — something “is not defined” ▶
You skipped a cell or ran cells out of order. Do Runtime → Restart and run all to re-run everything from the top. Cells depend on each other — if you skip the one that defines client or calculator, later cells won't work.
🔑 API key not found or Invalid API key ▶
Go back to Step 1, Part B. Check: (1) secret name is exactly GOOGLE_API_KEY (all caps, underscores), (2) “Notebook access” is toggled ON, (3) you pasted the full key with no extra spaces. If it still fails, create a new key in AI Studio.
⏳ Quota exceeded or 429 Resource exhausted ▶
The free API has rate limits. Wait about 60 seconds and try again. If it keeps happening, wait a few minutes. The limit resets quickly — don't panic.
🛠️ My custom tool doesn't get called by the agent ▶
Two common causes: (1) Your tool description is too vague — make it specific about when Gemini should use it. (2) The function name in my_functions doesn't match the name in your tool definition. They must be the exact same string.
🐛 I get a syntax error in my Python function ▶
Copy your code and the error message, paste it into ChatGPT or Claude, and say: “I'm getting this error in my Python function. Can you fix it?” AI tools are great at debugging syntax errors — that's literally what they're for.
🗑️ I accidentally deleted a cell ▶
Press Ctrl+Z (or Cmd+Z on Mac) to undo. If that doesn't work, try Edit → Undo in the Colab menu. Worst case: go back to the original notebook link, save a new copy, and redo the parts you lost.
